Extrareality API Client
=======

Это библиотека HTTP-клиента для работы с Extrareality. Она не обязательна к использованию, просто может сэкономить время.

Далее описание самого API.

Примеры в конце документа.

Подробнее об API
=======

Для многих запросов необходимо использовать обязательные параметры. Если они не обазятельны для каких-то отдельных методов, об этом сразу будет сказано.

##### Описание обязательных параметров:

* datetime - дата и время в формате Y-m-d H:i:s (подробнее в описании отдельных методов)
* owner_id - ваш id квеструма (**не** тот, что в каталоге, нужно уточнить у нас)
* quest_id - id квеста, для которого нужно получить отзывы (по умолчанию такой же, как в урле, например у "Похищения" = 12, http://extrareality.by/quest/12), но можно использовать и внутренний ID на вашем сайте, тогда сообщите нам об этом
* signature - шифрованная подпись; как формируется, можно глянуть в методе [\Extrareality\Client::generateSignature()](https://github.com/riente/extrareality-api/blob/78f8d4d6e535489e6bea80bc3391dfdd78e7c991/src/Client.php#L162)

##### Опциональные параметры (для любого запроса):

* source - источник запроса; это пригодится, если вы сотрудничаете с кем-то, у кого также реализован наш API, и вы хотите отслеживать, кто именно приводит вам клиентов (от нас source будет extrareality)

### Быстрые ссылки к описанию методов

**Для полноценной работы по бронированию** вам достаточно реализовать методы **забронировать**, **отменить бронь**, **массив цен** и **получить список броней по дате**; не обязательно, но очень желательно **проверить есть ли бронь**. Остальные по желанию, но они обеспечивают более чёткое взаимодействие. 

Если вы работаете с Миром Квестов и настраивали их API, то **ещё более простой вариант** для вас - [немного доработать](#доработка-мира-квестов) выдачу расписания, и не реализовывать остальные методы нашего API.

Вы должны предоставить нам URL с вашего сайта, по которому реализован наш API, и на который мы будет слать запросы, например https://your-site.by/xr-api

Для каждого метода указан свой endpoint, к примеру, если у метода "Забронировать" endpoint _/book_, то отправлять запрос на бронь мы будем по адресу https://your-site.by/xr-api/book

Мы отправляем запросы только на следующие endpoint-ы: 
/book, /check, /cancel, /prices.

Нам вы можете слать и другие запросы, описание которых приведено ниже. Корневой URL нашего API - https://extrareality.by/api, к которому добавляются _/book_, _/cancel_, прочие.

* **[Забронировать](#забронировать)** - обязательно
* **[Массив цен](#массив-цен)** - обязательно
* [Обновить расписание](#обновить-расписание) - не обязательно, но полезно, если вы меняете у себя расписание и/или цены и хотите, чтобы всё сразу же у нас обновилось
* **[Отменить бронь](#отменить-бронь)** - обязательно отправлять на наш сервер, если у вас отмена, но от нас обрабатывать не обязательно. Сами мы ничего не отменяем, такой запрос шлётся только если вы пользуетесь нашим кабинетом и делаете отмены через него
* **[Получить список броней по дате](#получить-список-броней-по-дате)** - обязательно
* [Получить список отзывов](#получить-список-отзывов)
* [Проверить, есть ли бронь](#проверить-есть-ли-бронь) - не обязательно, но **очень желательно**

## Получить список отзывов

Можно использовать не чаще раза в 30 минут.

Адрес для запроса будет такой:
https://extrareality.by/api/reviews?datetime=...&quest_id=...&owner_id=...&signature=...

Метод GET

```
Возвращать будет application/json, массив в виде
{
    REVIEW_ID: {
        "datetime": ...,
        "name": ..., // имя, кто оставил
        "text": ..., // текст отзыва
        "rating": ... // средняя оценка
    },
    REVIEW_ID2: {...}
}
```

Среди [обязательных параметров](#описание-обязательных-параметров): datetime содержит время, начиная с которого получать отзывы.

**Опциональные параметры:**

* newer_than_id - если указан, то отзывы берутся не начиная с даты, а начиная с этого id отзыва (т.е. при получении списка, можно максимальный id сохранять, а на следующий день получать начиная с него)
* quantity - количество отзывов для получения (если меньше 1, то возьмется 50)
* rating_threshold - не ниже какой средней оценки брать отзывы (7.5, 8 и т.д.)


Забронировать
---

https://extrareality.by/api/book

Метод: POST

Endpoint: /book

Параметры те же, как [описано выше](#описание-обязательных-параметров), в datetime должно быть время, на которое ставится бронь в формате 2015-11-06 22:00 или 2015-11-06 22:00:00. Также можно передавать параметры "name", "phone", "email", "comment", "players_num" (количество игроков). Соответственно, если запрос идет с нашей стороны на ваш сайт, то эти параметры тоже могут передаваться, и нужно настроить их обработку.

Возвращает код 200 и текст ОК, если все хорошо, или 400 и текст ошибки, если что-то не так.

Отменить бронь
---

https://extrareality.by/api/cancel

Метод: POST

Enpoint: /cancel

Параметры [те же](#описание-обязательных-параметров).
Возвращает то же, что и book.

Проверить, есть ли бронь
---

https://extrareality.by/api/check

Метод: POST

Endpoint: /check

Параметры [те же](#описание-обязательных-параметров).
Возвращает массив в json: если свободно, то [booked => 0], 
если занято, то [booked => 1, name => "Some name", phone => "Phone"]

Получить список броней по дате
---

https://extrareality.by/api/schedule

Метод: GET

Endpoint: /schedule

Среди [обязательных параметров](#описание-обязательных-параметров): datetime пишется в полном формате, но учитывается только дата, т.е. время можно написать любое.

Можно передать необязательный параметр days (по умолчанию 1), если вам нужно расписание на несколько дней вперед, начиная с указанной даты (но не более 30). Например: https://extrareality.by/api/schedule?days=7

Возвращает массив в json, в котором ключ это точные дата и время, а значение - массив в виде [time => ..., name => ..., phone => ...]

Пример:

```json
{
    "2018-05-01 15:00:00": {"time": "15:00", "name":"-", "phone": "-"},
    "2018-05-01 17:00:00": {"time": "17:00", "name": "Ермолай", "phone": "1234567"},
    "2018-05-01 19:00:00": {"time": "19:00", "name":"-", "phone": "-"}
}
```

Обновить расписание
---

https://extrareality.by/api/update_schedule?quest_id=...&owner_id=...&datetime=...&signature=...

Метод: POST

Endpoint: /update_schedule

В случае ошибок или нестыковок будет возвращен текст ошибки с кодом 400. Если запрос обработан успешно, то "ОК" со статусом 200.

В этом случае [параметры](#описание-обязательных-параметров) datetime (любая, хоть текущая дата), owner_id, quest_id и signature передаются в query string, а в теле запроса вам необходимо передать json encoded массив с данными в следующем формате:

```json
{
  "priceTemplates": {
    "Цена по будням": [
      {
        "name": "2 человека",
        "price": 150.0,
        "currency": "BYN"
      },
      {
        "name": "3 человека",
        "price": 170.0,
        "currency": "BYN"
      }
    ],
    "Цена по выходным": [
      {
        "name": "2 человека",
        "price": 160.0,
        "currency": "BYN"
      },
      {
        "name": "3 человека",
        "price": 180.0,
        "currency": "BYN"
      }
    ]
  },

  "slotTemplates": {
    "Будни": [
      {
        "slotTime": "15:00",
        "priceTemplate": "Цена по будням",
        "comments": ""
      },
      {
        "slotTime": "16:30",
        "priceTemplate": "Цена по будням",
        "comments": "Если одного из вас зовут Тедом, будет скидка"
      }
    ],
    "Выходные": [
      {
        "slotTime": "15:00",
        "priceTemplate": "Цена по выходным",
        "comments": ""
      },
      {
        "slotTime": "16:30",
        "priceTemplate": "Цена по выходным"
      },
      {
        "slotTime": "18:30",
        "priceTemplate": "Цена по выходным"
      }
    ]
  },

  "schedule": {
    "1": "Будни",
    "2": "Будни",
    "6": "Выходные",
    "2018-05-09": "Выходные",
    "7": false,
    "2018-05-10": false
  }
}
```

Пояснения: вы передаете объект (ассоциативный массив) с тремя другими массивами.
* __priceTemplates__ - ключ это имя шаблона цен (должно быть уникальным). Шаблонов может быть много, и для разных слотов вы можете применять разные шаблоны цен. Например в слоты до обеда вы хотите отображать одни цены, а после - другие.
* __slotTemplates__ - здесь также ключ является именем шаблона и должен быть уникальным. Значение после каждого ключа - это массив объектов слотов, в каждом из которых указано время и шаблон цен, применяемый к нему. Таких шаблонов у вас тоже может быть несколько, например в будние дни у вас одно расписание, а в выходные вы работаете до поздней ночи.
* __schedule__ - непосредственно правила вашего расписания. Если ключ это цифра от 1 до 7, значит задается правило для дня недели (1 - понедельник, 2 - вторник и т.д.). Если строка в формате YYYY-MM-DD, то для конкретной даты. Значение может содержать имя шаблона слотов или false (если в этот день вы не работаете совсем). Стоит иметь в виду, что можно одновременно передать правила и для дня недели, и для даты, но правило даты будет иметь приоритет в соответствующий день. Также, если какие-то дни недели у вас пропущены, то они по умолчанию будут приниматься как false, т.е. нерабочий день (например, вы прислали правила для 1, 2, 3, 6 и 7, четверг и пятница получаются выходными).

__Внимание!__ В именах шаблонов можно использовать только буквы, цифры, точку, запятую, точку с запятой, дефис, подчеркивание и пробел.

Пример отправки запроса с помощью cURL:

```php
$params = [
	'quest_id' => $questId,
	'owner_id' => $ownerId,
	'signature' => $this->generateSignature($datetime, $questId),
];

$url = 'https://extrareality.by/api/update_schedule';
$url .= '?'.http_build_query($params);
$curl = curl_init($url);

curl_setopt($curl, CURLOPT_FOLLOWLOCATION, true);
curl_setopt($curl, CURLOPT_HEADER, false);
curl_setopt($curl, CURLOPT_SSL_VERIFYPEER, false);
curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);
curl_setopt($curl, CURLOPT_CUSTOMREQUEST, 'POST');
curl_setopt($curl, CURLOPT_POSTFIELDS, $body);
curl_setopt($curl, CURLOPT_HTTPHEADER, [
	'Content-Type: application/json',
	'Content-Length: '.mb_strlen($body, 'utf-8')
]);

$result = curl_exec($curl);
$info = curl_getinfo($curl);
$code = $info['http_code'];

curl_close($curl);
```

Массив цен
---

Метод: GET

Endpoint: /prices

Здесь не нужно отправлять какие-то запросы и передавать параметры.

Это более простой вариант реализации [предыдущего метода](#обновить-расписание). По адресу с окончанием /prices вы должны разместить на своем сайте файл/скрипт, который будет отдавать массив временных слотов на месяц вперед (срок на ваше усмотрение).

Мы передаем параметры signature, quest_id, owner_id и datetime, но datetime будет пустой. Вы должны учитывать quest_id, и можете проверять signature при желании.

Каждый слот - это объект, содержащий следующие свойства:
* date (Y-m-d)
* time (H:i)
* prices (массив цен, в котором _ключ_ - это условие, _значение_ - цена).

Пример:

```json
[
  {
    "date": "2018-11-14",
    "time": "10:00",
    "prices": {
      "2 человека": 15,
      "3 человека": 20,
      "4 человека": 25,
    }
  },
  {
    "date": "2018-11-14",
    "time": "12:00",
    "prices": {
      "2 человека": 15,
      "3 человека": 20,
      "4 человека": 30,
    }
  },
  ...
]
```

Доработка Мира Квестов
---

Метод: GET

Endpoint: на ваше усмотрение

Для бронирования оставляете всё как есть и сообщаете нам необходимые данные, например md5-код и URL endpoint-а.

Что касается расписания, вы выдаете список всех ваших слотов примерно на месяц вперед так же, как и для МК, но добавляете в каждый из них поле "extraPrices", которое содержит массив. В нём: ключ - условие, значение - цена. На работу с МК это никак не влияет. Пример:

```
[
   {
       "date": "2016-05-05",
       "time": "18:30",
       "is_free": true,
       "price": 3000,
       "extraPrices": {
           "2 человека": 80,
           "Больше 2 человек": 110
       }
   },
   {
       "date": "2016-05-05",
       "time": "20:00",
       "is_free": false,
       "price": 3500,
       "extraPrices": {
           "2 человека": 70,
           "Больше 2 человек": 100
       }
   },
]
```

Примеры
=======

К примеру на вашем сайте происходит бронь, и вы хотите "уведомить" об этом Extrareality.

```php
use Extrareality\Client;

$config = [
    'secret' => 'somesecretkey', // его вам нужно узнать у нас
    'ownerId' => 123, // ID вашего квеструма в нашей базе (тоже спросите у нас)
];

$questId = 5; // ID квеста в вашей базе (если он отличается от нашего, имеет смысл сообщить его нам, тогда мы будет отправлть именно его)
$datetime = '2017-04-20 09:00:00'; // время, на которое бронируется квест

try {
    $client = new Client($config['ownerId'], $config['secret'], $questId);
    // Optionally you can set a custom source or API URL
    // $client->setSource('yoursitename');
    // $client->setApiUrl('https://partner.site/api/');
    $client->book($datetime, $questId);
} catch (Exception $e) {
    // handle possible exceptions
}
```

Также вы хотите получать и обрабатывать запросы от Extrareality. У вас есть контроллер по адресу http://somesite.ru/api и все запросы (вроде /api/book, /api/check, /api/schedule и прочие) направляются на него.

```php
use Extrareality\ApiRequest;
use Extrareality\Response\ScheduleResponse;

$config = [
    'secret' => 'somesecretkey', // его вам нужно узнать у нас
];

$request = new ApiRequest($config['secret']);

try {
    $api = new ApiRequest($config['secret']);
    if ($api->isBooking()) {
        // регистрируем бронь у себя в системе
    } elseif ($api->isCancel()) {
        // ...
    } elseif ($api->isSchedule()) {
        $response = new ScheduleResponse();
        
        // Pseudocode
        $myBookings = getBookingRecords();
        foreach ($myBookings as $booking) {
            $response->addBookingToSchedule(
                new \DateTime($booking->getTimestamp()),
                $booking->getName(),
                $booking->getPhone()
            );
        }

        $response->prepare();

        http_response_code($response->getCode());
        header('Content-Type: '.$response->getContentType().'; charset=utf-8');
        echo $response->getMessage();
        exit;
    } elseif ($api->isCheck()) {
        // ...
    }
} catch (Exception $e) {
   // handle
}
```
